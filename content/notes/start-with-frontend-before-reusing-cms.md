## 원래 계획

처음에는 Criminal Law Archive의 CMS를 복제해 새로운 기록 사이트에 연결하려고 했다.

이미 관리자 기능과 콘텐츠 관리 구조가 있었기 때문에, 새로 만드는 것보다 재사용하는 것이 효율적이라고 생각했다.

## 달라진 판단

하지만 CMS를 옮기는 작업 자체가 하나의 큰 프로젝트가 되기 시작했다.

아직 어떤 필드가 오래 유지될지, 실제로 어떤 방식으로 기록할지도 충분히 확인되지 않은 상태였다.

이 시점에 CMS 구조부터 고정하면 이전 시스템의 복잡성을 그대로 가져오거나, 실제 사용과 맞지 않는 스키마를 다시 수정할 가능성이 컸다.

## 결정

기존 CMS를 바로 재사용하지 않고, Working Notes에 필요한 화면과 기록 흐름을 프론트에서 먼저 만들기로 했다.

우선 작은 Markdown 구조로 기록을 작성하고 다음을 확인한다.

- 반복해서 사용하는 필드가 무엇인지
- 이미지가 실제로 필요한지
- type과 tags를 어떻게 사용하는지
- 작성과 수정 과정에서 무엇이 불편한지
- 검색이나 필터가 필요한지

## 다음에 확인할 것

몇 주간 실제 기록을 작성한 뒤 Markdown 관리가 불편해지면 CMS 연결을 다시 검토한다.

그때는 예상이 아니라 실제 기록 데이터를 기준으로 스키마와 관리자 기능을 결정할 수 있다.
